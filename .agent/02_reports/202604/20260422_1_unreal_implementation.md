# Unreal Engine V1 Implementation Plan
**Date**: April 22, 2026
**Reference Document**: `docs/17_its_getting_unreal.md`

## 1. Goal Description
Implement the V1 Unreal Engine Lighting Node. This incorporates backend DMX fan-out routing, explicit Unreal DMX input port configurations, Python headless scene ingestion preserving true architectural meshes, and the replacement of the CaptainPad WebGL WebView with a genuine Pixel Streaming client.

## 2. Implementation Sequence

### Phase 1: DMX Fan-Out Modifications
The engine must send frames to both the physical hardware and the Unreal Engine listener.
#### [MODIFY] `marsin_engine/lib/sacn_output.js`
*   Refactor current single-unicast logic into an explicit **multi-unicast fan-out** array loop `destinations[]`.
*   Ensure zero latency loss between broadcasting the post-processed DMX to `10.x.x.x` physical networks vs local Unreal Engine NIC interfaces.

### Phase 2: Unreal Engine Input Architecture
Unreal cannot receive anything if its ports are closed. The DMX protocol connection must be a first-class citizen.
#### [NEW] `Unreal DMX Project Configuration`
*   Configure the **sACN Input Port Config** correctly defining: NIC IP, Universe Base/Count ranges matching the `patches.yaml`, and mapping Priority models properly within Unreal Editor / Project settings. 

### Phase 3: Unreal Headless Python Ingestion
Transforming the `.yaml` structures into the mapped Unreal DMX Library and Visual hierarchies.
#### [NEW] `simulation/unreal/scripts/ingest_scene.py`
*   **Scene Merging**: Combine and parse `common.yaml`, `scene_config.yaml`, `patches.yaml`, and `cameras.yaml`.
*   **Fixture Type Parsing**: Read `fixture_definition_registry.js` schemas to correctly determine properties (UkingPar vs ShehdsBar) when instantiating Unreal DMX `FixtureTypes`.
*   **DMX Library Aliasing**: If a patch mapping duplicates an address (e.g., U1:A512 foggers), the Python script generates exactly ONE synthetic `FixturePatch` and maps multiple spawned Blueprints to the singular patch.
*   **Geometry Rule**: If an object declares actual DMX attributes (e.g. standard fixtures and updated `LED Strands`), they map as active listeners (binding `OnFixturePatchReceivedDMX`). If an element explicitly lacks patching (`patch: null` like Icebergs), it MUST still be spawned physically as **Read-only Static Geometry** so the scene layout structurally remains intact, avoiding visual divergence from the authored plan.

### Phase 4: CaptainPad UI Monitor Replacement
The Web UI must embed a true video stream, not just a status hook.
#### [MODIFY] `CaptainPad/app/(tabs)/monitor.tsx`
*   Replace the raw generic `http://host:6969/simulation` WebView iframe.
*   Implement a direct Unreal **Pixel Streaming UI WebRTC Client wrapper**. This will accept streaming playback generated by the Engine Server.
#### [MODIFY] `marsin_engine/lib/api_server.js`
*   Build the `GET /status` API returning active context constraints: `{ activeScene, activeModel, activePattern, unrealState }`.
*   `unrealState` ("offline" | "loading" | "streaming") determines CaptainPad's offline/fallback rendering strategy explicitly.
